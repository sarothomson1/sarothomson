My Drawing Application

My aim is to a canvas which a user can utilise to create artwork

I have made eight sprints to achieve this. Over each sprint I have developed the program to create a potentially shippable product

Below is the planning, testing and finished products of each of these sprints

Relevent Implications

Future Proofing

My program is designed so new features can be added easily and old ones removed
There are multiple aspects my code that allow for simple redesigning of the program
My program elements (eg.buttons, colours, shapes etc.)have all been programmed as objects, means each is contained in a self-contained modular piece of code, that takes a defined set of parameters to work. Many of these take basic mouse data and colour information
All my objects are built to a similar structure and compulsorily contain an update function. This means they can appear on the interface using the update call
The management of most of the objects happens in a central place (main.js). This is to allow efficient addition and removal of current elements, and have a logical space to add new features
I have also thoroughly and precisely commented my code, so it can be easily interpreted by other users

Functionality

My program fucntions well - is free of bugs and loads quickly
This has been achieved through creating my code in pure jaavscript, has no graphics so loads very quickly. I have tested by using a live web server to store my versions, and have observed loading performance. To ensure the program works well I have systematically tested it. I tested user interactions - position of clicking and direction of dragging. This has also included careful checks around boundaries. The use of Maths.abs function was also discovered after the testing failures drawing circular shapes. All new components were tested prior to being added to the main program At various stages in sprints, I used console logging to analyse boundary test functions and to locate problems with bugs appeared.

Examples of trialling and decision making

Trialing different options for allowing the users to draw circles on screen, giving circular or rectangular guides Effectively using information from testing and trialling to improve the functionality of the outcome Finding situations were objects drew outside of the drawing area (in spite of having in-boundary parameters) led to the use of the clip() function.
